hashcode 与 equals 简介

  • equals()
    equals() 用于比较两个对象是否相等，它与 == 比较符有着本质上的区别
    在万物皆对象的Java体系中，系统把判断两个对象是否相等的权利交给了程序员，具体的措施是把equals()方法写到Object类中，并让所有的类都继承Object类，这样程序员就可以在自己的自定义类中重写equals()方法，从而实现自己的比较逻辑
  • hashcode()
    1. hashcode 的意思是哈希值，哈希值是经过哈希运算得到的结果，哈希函数能够保证相同的输入会得到相同的输出(哈希值)，但是不能够保证不同的输入总是会得到不同的输出(哈希冲突)
    2. 当输入的样本数量足够大时，是会产生哈希冲突的，也就是说不同的输入得到了相同的输出
    3. 抛开不同的输入产生相同的输出，就相同的输入能够产生相同的输出这点而言是很有价值的，它使得系统只需要简单的运算，在时间复杂度为O(1)的情况下就能得到数据的映射关系，根据这种特性，散列表应运而生。
    4. 一种主流的散列表实现是：用数组作为哈希函数的输出域，输入值经过哈希计算得到哈希值，然后根据哈希值，在数组中找到对应的存储单元，当发生冲突时，对应的存储单元以链表的形式保存冲突的数据

初始 hashcode 与 equals 的关系

  • 在大多数的编程实践中，归根结底会落实到数据的存取问题上。在汇编语言的时代，你需要老老实实地对每个数据操作编写存取语句。而随着时代发展到今天，我们都用方便灵活的高级语言编写代码，比如Java。Java以面向对象为核心思想，封装了一系列操作数据的API，降低了数据操作的复杂，但在我们对数据进行操作之前，首先要把数据按照一定的数据结构保存到存储单元中，否则操作数据将无从谈起，然而不同的数据结构有各自的特点，我们在存储数据的时候需要选择合适的数据结构进行存储。Java根据不同的数据结构提供了丰富的容器类，方便程序员选择合适自身业务的容器类进行开发。
  • 通过继承关系图我们看到Java的容器类被分为 Collection 和 Map 两大类，Collection 又被分成 List 和 Set。其中Map和Set都是不允许元素重复的，严格的来说map是不允许重复的键值。
  • 值得注意的是，Map 和 Set 的绝大多数实现类的底层都使用到了散列表结构

深入理解 hashcode 和 equals 之间的关系

  1. equals() 会有力不从心的时候
    上面提到的Set 和 Map 不允许存放重复的元素(key)，这边容器在存储元素时必须对元素做出判断：在当前容器中有没有和新元素相同的元素，如何去实现这一点呢，可能很容易就想到直接调用元素的equals()方法进行比较不就行了？这个方法确实可以，但是缺点也非常的明显，如果容器中存储的对象很少，这确实是个好主意，但是容器中存放的对象达到了一定的规模，要调用容器中所有对象的equals()方法和新元素进行比较，就不是一件很容易的事情了。就算 equals()方法的比较逻辑简单无比，总的来说也是一个时间复杂度为O(n)的操作

  2. hashcode() 小力出奇迹
    但在散列表的基础上，判断 “新对象是否和已存在对象相同” 就容易多了。由于每个对象都自带有hashcode()，这个hashcode将会用作散列表哈希函数的输入，hashcode经过哈希函数计算后得到哈希值，新对象会根据哈希值，存储到相应内存的单元。
    我们不妨假设两个相同的对象，hashcode() 一定相同，这么一来就体现哈希函数的威力了，由于相同的输入一定会产生相同的输出，于是如果新对象和容器中已经存在的对象相同，新对象计算出的哈希值和已存在对象的哈希值产生冲突，此时容器就能判断，这个新加入的元素在容器中已经存在，需要另外做处理，按照这个思路，如果这个元素计算出的哈希值和所对应的内存单元没有产生冲突，也就是没有重复的元素，那么它就可以直接插入，所以当运用hashcode()时，判断是否有相同元素的代价，只是一次哈希计算，时间复杂度为O(1)，这极大的提高了数据的存储性能

  3. 设计hashCode() 和equals()时约定的规则
    前面提到，当输入的样本足够大时，不相同的输入也会产生相同的输出，也就是形成哈希冲突。这么一来，原本我们设定的 如果产生冲突，就意味着两个对象相同 的规则被打破，因为产生冲突的很有可能是两个不同的对象！幸好除了hashcode()方法，还有一个equals()方法，也就是说两个不同的对象产生冲突后，我们可以用equals()方法进一步判断两个对象是否相同，此时equals()方法就相当重要了，这个情况下他必须能判定这两个对象是不相同的。

    • Java程序设计中一个重要的原则：
      如果两个对象是相等的，他们的equals()方法应该要返回true，他们的hashcode()需要返回相同的结果。
